Data Flow Examples

🎮 Teleop Command

  • User presses D-pad / moves joystick
  • controls.js computes velocity (vx, vy, vyaw)
  • Twist message published to /cmd_vel
  • rosbridge forwards via WebSocket to ROS2
  • Go2 controller executes movement

📍 Click-to-Navigate

  • User clicks destination on map canvas
  • Canvas pixel → world coordinates
  • PoseStamped published to /goal_pose
  • Nav2 plans path with obstacle avoidance
  • Robot follows planned trajectory

🗺️ Map & Localization

  • FAST-LIO2 builds 3D point cloud map
  • OccupancyGrid published to /map
  • rosbridge streams to browser via WebSocket
  • Canvas renders map + robot pose + LiDAR overlay
  • User saves map via REST → .yaml + .pgm

⚠️ Emergency Stop (Damp)

  • User clicks Damp button
  • POST /api/sport with api_id: 1001
  • Flask publishes to /api/sport/request
  • Sport controller receives Damp command
  • Robot immediately dampens all joints

Deployment Modes

💻

Jetson Mode (On-Robot)

Flask + rosbridge run directly on the Jetson Orin. Lowest latency, zero network dependency. Commands execute locally.

🌐

Remote Mode (PC/Laptop)

Flask on external PC, commands relayed via SSH to Jetson. Enables development and control from any device on the network.

⚡

Hybrid (Direct rosbridge)

Browser connects directly to rosbridge WebSocket on robot. Zero-latency teleop. Flask only for management APIs.
